# 基于深度学习与知识图谱的长江支流水质预测与可解释性分析

## 一、项目背景与问题

当前长江支流水质监测面临**三重困境**：

1. **时空稀疏**：自动监测站分布不均，数据采集间隔长，难以捕捉突发性水质恶化事件；
2. **预测滞后**：传统水质评价以"事后分析"为主，缺乏面向未来 24 小时的实时预警能力；
3. **黑箱困境**：现有深度学习模型虽精度可观，但"为什么预测水质会下降"无法向管理者解释清楚，严重制约了模型在实际决策中的落地。

本项目以**岷江流域**（长江上游最大支流）为研究区域，选取 **15 个自动监测站、9 万余条 4 小时级高频数据（2022–2025）**，构建了一套**"深度学习预测 — 知识图谱解释 — 交互可视化展示"**三位一体的技术方案，致力于让水质预测**既准确、又可懂、更可用**。

---

## 二、技术路线与创新点

### 整体架构

```
原始监测数据 → 数据清洗/4h重采样 → 特征工程(19维) → LSTM+Attention建模 → 知识图谱构建 → Streamlit可视化系统
```

### 创新点 1：LSTM + Attention 多步溶解氧预测

- **回看 72 小时、预测未来 24 小时**：以 18 个 4h 时间步为输入窗口，输出未来 6 个时间步（4h/8h/12h/16h/20h/24h）的溶解氧（DO）预测值；
- **19 维融合特征**：9 个原始水质参数（水温、pH、DO、电导率、浊度、高锰酸盐指数、氨氮、总磷、总氮）+ 4 个周期编码时间特征 + 6 个滚动趋势特征（DO 短/长期均值、波动幅度、变化速率、水温趋势）；
- **注意力机制**：模型自动学习"过去哪些时间步对当前预测最关键"，权重可直接用于可解释性分析；
- **逐站独立建模**：为 7 个重点站各训练专属模型，充分适应不同站点的水文特征差异。

### 创新点 2：领域知识图谱驱动的可解释性

不同于传统的"特征重要性排序"，本项目构建了覆盖 **8 类实体、13 类关系** 的领域知识图谱：

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| Station | 监测站点 | 渭门桥、都江堰水文站 |
| River | 河流水系 | 岷江干流、大渡河 |
| Parameter | 水质参数 | 溶解氧、水温、氨氮 |
| ModelResult | 模型预测结果 | R²=0.95 的渭门桥模型 |
| Event | 水质事件 | DO 低于 III 类标准预警 |
| WaterQualityClass | 水质类别 | GB 3838 I~V 类 |
| City | 所属城市 | 成都、乐山 |
| DataSource | 数据来源 | 国家地表水自动监测站 |

通过**"站点—河网—子流域—参数—事件—模型结果"**的语义关联，将置换特征重要性（Permutation Importance）和注意力权重与水环境领域知识融合，实现"模型说了算 → 领域专家也能看懂"的可解释转化。

### 创新点 3：交互式原型系统

基于 Streamlit 构建了面向非技术用户的 Web 原型系统：
- **地图总览**：Folium 地图点选 15 个站点，一目了然查看站点分布与水质状态；
- **24h 预测卡片**：每 4 小时一张卡片，以 GB 3838 水质类别色标直观展示未来水质趋势；
- **驱动因子 TOP5**：结合置换重要性与领域知识，自然语言解释"为什么预测会这样"；
- **回测验证曲线**：Plotly 交互图表，对比实际值与预测值，增强用户信任度。

## 三、核心实验结果

### 3.1 七站预测性能总览

| 站点 | MAE (mg/L) | RMSE (mg/L) | R² | MAPE (%) | 训练耗时 |
|------|-----------|-------------|-----|---------|---------|
| **渭门桥** | **0.171** | **0.229** | **0.949** | **1.77** | 32s |
| 都江堰水文站 | 0.273 | 0.596 | 0.536 | 3.06 | 37s |
| 岳店子下 | 0.638 | 0.895 | 0.737 | 6.42 | 68s |
| 悦来渡口 | 0.288 | 0.412 | 0.767 | 3.14 | 111s |
| 月波 | 0.237 | 0.309 | 0.786 | 2.55 | 29s |
| 李码头 | 0.339 | 0.475 | 0.809 | 3.44 | 93s |
| 姜公堰 | 0.718 | 0.920 | 0.809 | 7.72 | 172s |

> **渭门桥站达到 R²=0.949、MAPE 仅 1.77%**，即使在 24 小时远期预测步上仍保持 R²=0.946 的高精度。7 个站点中 **5 个站点 R² 超过 0.76**，整体预测误差在环境监测可接受范围内。

### 3.2 多步预测精度衰减特性

以渭门桥站为例，模型精度随预测时间步推移呈现**平缓衰减**，表明注意力机制有效缓解了长步预测中的误差累积问题：

| 预测步 | 预测时长 | R² | MAPE (%) |
|-------|---------|-----|---------|
| Step 1 | 4h | 0.957 | 1.61 |
| Step 2 | 8h | 0.952 | 1.73 |
| Step 3 | 12h | 0.950 | 1.77 |
| Step 4 | 16h | 0.946 | 1.84 |
| Step 5 | 20h | 0.946 | 1.83 |
| Step 6 | 24h | 0.946 | 1.85 |

> 从 4h 到 24h，R² 仅下降 0.011，MAPE 仅上升 0.24 个百分点，展现出优异的远期预测稳定性。

### 3.3 可解释性发现

通过置换特征重要性与注意力权重分析，揭示了具有水环境科学意义的规律：

- **DO 变化速率（DO_diff）** 在多数站点中位列最重要特征 TOP3，表明短期趋势对预测至关重要；
- **水温及其趋势** 在姜公堰等上游站点贡献突出，符合"水温升高→DO 溶解度下降"的物理机制；
- **注意力权重呈指数递增分布**，最近 4h（t-1）权重普遍为最远时间步（t-18）的 10~20 倍，但远期信息仍贡献约 10% 的总权重，验证了 72h 回看窗口的必要性；
- **不同站点驱动因子差异显著**：上游站（姜公堰）受水温主导，中下游站（悦来渡口）受浊度和营养盐影响更大，体现了河流沿程水质演变的空间异质性。

## 四、项目结构

```
├── data/
│   └── processed/
│       ├── minjiang_4h.csv          # 核心数据集：15站×4h级水质时序（~9万条）
│       └── minjiang_stations.csv    # 站点元信息（经纬度、河流、数据时段等）
├── src/
│   ├── data_processing/             # 数据清洗与预处理管线
│   │   ├── prepare_minjiang.py      #   原始数据→4h重采样+周期编码
│   │   ├── merge_data.py            #   多源数据合并
│   │   ├── clean_data.py            #   异常值清洗
│   │   ├── aggregate_daily.py       #   日尺度聚合
│   │   └── select_stations.py       #   重点站筛选
│   ├── model/                       # 深度学习建模模块
│   │   ├── lstm_model.py            #   LSTM+Attention 模型定义
│   │   ├── dataset.py               #   数据集构建（19维特征/滑动窗口/防泄漏标准化）
│   │   ├── train.py                 #   逐站训练（早停/学习率衰减）
│   │   ├── inference.py             #   推理与24h预测卡片生成
│   │   └── feature_importance.py    #   置换特征重要性与注意力权重提取
│   ├── knowledge_graph/             # 知识图谱模块
│   │   ├── build_kg.py              #   KG 构建（NetworkX→JSON）
│   │   ├── kg_schema.json           #   图谱 schema（8类实体/13类关系）
│   │   └── kg_data.json             #   图谱实例数据
│   └── app/                         # 可视化展示模块
│       ├── app.py                   #   Streamlit 主页（地图/KPI/状态表）
│       └── pages/
│           └── 1_📊_站点分析.py      #   站点详情页（预测/解释/回测）
├── models/                          # 训练产物
│   ├── all_results.json             #   全站训练指标汇总
│   ├── all_feature_importance.json  #   全站特征重要性汇总
│   ├── forecasts.json               #   最新24h预测结果
│   └── [7个站点子目录]/              #   各站模型权重/预测/重要性
├── requirements.txt                 # 依赖清单
├── run_train.py                     # 一键训练入口
└── 启动网站.bat                      # 一键启动可视化系统
```

## 六、快速开始

> 说明：GitHub 演示仓库默认保留可视化系统所需的预计算结果文件（如 `models/forecasts.json`、各站点回测结果等），因此下载后可直接运行展示页面；原始大体量数据 `data/processed/minjiang_4h.csv` 默认不随仓库提供。

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 训练模型（可选，已有预训练结果）
python run_train.py

# 3. 启动可视化系统
streamlit run src/app/app.py --server.port 8501
# 或直接双击 启动网站.bat
```

## 七、应用价值与展望

**当前成果**：已实现岷江流域 7 个重点站的 24 小时溶解氧预测，最优站点 R² 达 0.949，并通过知识图谱将模型输出转化为管理者可理解的自然语言解释。

**未来方向**：
1. **空间扩展**：从岷江单一流域推广至长江全流域，纳入更多支流与断面；
2. **多指标预测**：从单一 DO 扩展至氨氮、总磷等多指标联合预测；
3. **图神经网络**：利用知识图谱中的河网拓扑信息，引入 GNN 建模站点间的空间关联；
4. **实时接入**：对接国家地表水自动监测数据接口，实现从"离线分析"到"在线预警"的升级。
